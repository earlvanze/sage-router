Sage Router Terms

Effective June 19, 2026. These terms cover the hosted Sage Router web app, public API edge, account management, billing, generated API keys, quotas, analytics, and related hosted routing control-plane services.

Service Boundary

Sage Router sells routing infrastructure, account management, API key management, usage quotas, analytics, reliability tooling, support, and optional private deployment help. Sage Router does not grant unauthorized access to third-party model providers, does not promise to bypass provider terms, and does not pool customer provider accounts as a default service.

Accounts and API Keys

  • You are responsible for activity under your Sage Router account and generated sk_sage_* API keys.
  • Keep API keys confidential. Revoke keys immediately if they are exposed or no longer needed.
  • We may suspend or disable accounts, keys, or routes that create abuse risk, security risk, payment risk, infrastructure harm, or legal risk.

Provider Access

You are responsible for ensuring that any model provider, local model, subscription, API key, cloud account, or proxy endpoint you connect to Sage Router is authorized for your use. Provider terms, license terms, rate limits, content rules, and billing obligations remain between you and the provider unless a separate written agreement says otherwise.

Billing and Plans

Hosted plans may include monthly request quotas, request-per-minute limits, billing portal access, analytics, and support. Stripe checkout and billing portal flows are the primary self-service billing paths. Crypto or manual settlement may be available by agreement and may require manual activation.

Reliability

Sage Router is built for health-aware fallback and CDN-style routing across available installations, but no hosted service is guaranteed to be uninterrupted. Public status and route health are provided for operational visibility, not as a guarantee of model provider availability.

Acceptable Use

Your use must follow the Acceptable Use Policy. Do not use Sage Router to attack, overload, scrape, evade provider controls, violate third-party terms, or route unlawful content or traffic.

Privacy

The Privacy Policy describes hosted account data, billing data, generated-key metadata, usage counters, and privacy-safe route telemetry. Sage Router is designed not to store prompts, message bodies, provider credentials, OAuth tokens, API keys, or raw provider responses in launch analytics.

Changes

We may update hosted terms, plans, limits, security controls, or service behavior as the product matures. Material changes should be reflected on this page or in customer-facing release notes.
