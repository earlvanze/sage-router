Copyable recovery bundle

Paste this after the key exists.

The bundle is intentionally safe to copy because it uses a placeholder key. Replace it with the raw generated key shown once in the account page.

# Sage Router setup-key recovery
# 1. Finish the key using the same email or OAuth identity:
# https://app.sagerouter.dev/login.html?plan=pro&start=create_key&utm_source=setup-key-recovery&utm_medium=recovery&utm_campaign=signup_to_key_recovery&auth=email

export OPENAI_BASE_URL=https://api.sagerouter.dev/v1
export OPENAI_API_KEY=sk_sage_your_key_here
export OPENAI_MODEL=sage-router/frontier

# 2. Verify the generated key before wiring an agent:
curl -i "$OPENAI_BASE_URL/models" \
  -H "Authorization: Bearer $OPENAI_API_KEY"

What a 401 means

The API is fail-closed by design.

  1. Anonymous model traffic is blocked, so /v1/models returns 401 without a valid generated key.
  2. The JSON body includes keyRecoveryUrl, accountUrl, pricingUrl, openaiBaseUrl, and apiKeyPrefix for safe debugging.
  3. Do not paste prompts, provider credentials, OAuth tokens, generated API keys, private keys, session cookies, or raw provider responses into public support channels.
